Ejercicios de Javascript agrupados por Unidades Didácticas

UD1. Selección de arquitecturas y herramientas

¿Por qué es importante? Entender cómo funciona la web y las herramientas que usamos es clave para cualquier desarrollador.
RA/CE: Caracterizar ejecución cliente/servidor, conocer herramientas y la integración con HTML.
Ejemplo real: Saber cuándo tu código JS se ejecuta en el navegador o en el servidor te ayuda a elegir la mejor solución para una web interactiva.

  • L1. Modelo de ejecución en cliente vs. servidor
    ¿Qué ocurre cuando visitas una web? Descubre la diferencia entre lo que ocurre en tu navegador y en el servidor.
  • L2. Integración JS–HTML (etiquetas, carga diferida)
    Aprende a insertar y cargar JS correctamente en tus páginas. Ejemplo real: Evita errores de scripts que se ejecutan antes de que exista el HTML.
  • L3. Herramientas (devtools, linters, control de versiones) y librerías/frameworks
    Descubre cómo las herramientas profesionales te ayudan a programar mejor y más rápido. Ejemplo real: Usar la consola del navegador para depurar errores.

UD2. Sintaxis del lenguaje

¿Por qué es importante? Dominar la sintaxis de JS es la base para crear cualquier aplicación.
RA/CE: Uso de variables, operadores, decisiones, bucles, documentación.
Ejemplo real: Automatizar tareas, validar datos de usuario o crear juegos sencillos.

  • L1. Variables, ámbito y tipos; conversiones
    Aprende a guardar y transformar datos. Ejemplo real: Convertir la entrada de un formulario en número para calcular un precio.
  • L2. Operadores y expresiones
    Realiza cálculos y toma decisiones en tu código. Ejemplo real: Calcular descuentos o sumar puntuaciones en un juego.
  • L3. Control de flujo: sentencias, decisiones y bucles
    Controla el orden de ejecución y repite acciones. Ejemplo real: Recorrer una lista de productos para mostrarlos en pantalla.
  • L4. Comentarios, pruebas y documentación
    Documenta y prueba tu código para que sea comprensible y fiable. Ejemplo real: Añadir comentarios para que otros entiendan tu función de validación.

UD3. Objetos predefinidos y entorno navegador

¿Por qué es importante? Los objetos nativos y el entorno del navegador son la base para interactuar con la web.
RA/CE: Identificar objetos, cambiar aspecto/documento, interacción usuario, cookies, depuración.
Ejemplo real: Mostrar la fecha y hora actual en una web, guardar preferencias del usuario o depurar errores en el navegador.

  • L1. Objetos nativos (String, Number, Date, Math, Error…)
    Utiliza los objetos que JS te ofrece de serie. Ejemplo real: Calcular la edad de una persona a partir de su fecha de nacimiento.
  • L2. BOM: ventana, historial, temporizadores; generar HTML desde JS
    Interactúa con la ventana del navegador y genera contenido dinámico. Ejemplo real: Mostrar un mensaje tras unos segundos o navegar entre páginas.
  • L3. Almacenamiento Web (cookies, localStorage, sessionStorage)
    Guarda información en el navegador. Ejemplo real: Recordar el usuario logueado o las preferencias de idioma.
  • L4. Depuración en navegador
    Aprende a encontrar y corregir errores usando las herramientas del navegador. Ejemplo real: Usar breakpoints para analizar el flujo de un formulario.

UD4. Arrays, funciones y objetos del usuario

¿Por qué es importante? Crear tus propias funciones, arrays y objetos te permite resolver problemas reales y estructurar tu código.
RA/CE: Crear/usar funciones, arrays y objetos definidos por el usuario.
Ejemplo real: Gestionar una lista de tareas, calcular estadísticas o modelar usuarios en una app.

  • L1. Funciones (declaración, expresión, arrow, parámetros)
    Define y usa funciones para reutilizar código. Ejemplo real: Crear una función para calcular el IVA de un producto.
  • L2. Arrays y operaciones agregadas (map, filter, reduce)
    Trabaja con listas de datos y realiza operaciones complejas de forma sencilla. Ejemplo real: Filtrar productos por precio o sumar cantidades.
  • L3. Objetos: propiedades, métodos, patrones básicos
    Crea tus propios tipos de datos. Ejemplo real: Modelar un usuario con nombre, email y métodos para actualizar su perfil.
  • L4. Diseño y depuración
    Organiza y mejora tu código para que sea mantenible. Ejemplo real: Refactorizar una función larga en varias más pequeñas y probarlas.

UD5. Interacción: eventos y formularios

¿Por qué es importante? La interacción con el usuario es la esencia de la web moderna.
RA/CE: Capturar/utilizar eventos, validar formularios, usar regex, probar/documentar.
Ejemplo real: Validar un formulario de registro o mostrar mensajes cuando el usuario interactúa con la página.

  • L1. Modelo de eventos (captura/burbujeo, addEventListener)
    Gestiona eventos de usuario de forma eficiente. Ejemplo real: Detectar cuándo el usuario hace clic en un botón y responder a ello.
  • L2. Formularios: lectura, validación y envío; prevención de acciones
    Lee y valida datos de formularios antes de enviarlos. Ejemplo real: Evitar que se envíe un formulario si hay errores.
  • L3. Expresiones regulares para validación robusta
    Valida datos complejos como emails o contraseñas. Ejemplo real: Comprobar que un email tiene el formato correcto.

UD6. DOM

¿Por qué es importante? El DOM es la puerta de entrada para modificar cualquier página web desde JS.
RA/CE: Identificar objetos del modelo, acceder/alterar estructura, asociar eventos, independencia de capas.
Ejemplo real: Añadir elementos a una lista sin recargar la página o cambiar el estilo de un elemento al pulsar un botón.

  • L1. Recorrido y selección (querySelector, colecciones)
    Selecciona y recorre elementos del DOM. Ejemplo real: Resaltar todos los enlaces de una página.
  • L2. Creación y modificación dinámica (createElement, classList, estilos)
    Crea y modifica elementos sobre la marcha. Ejemplo real: Añadir tareas a una lista interactiva.
  • L3. Delegación de eventos y accesibilidad
    Gestiona eventos de forma eficiente y accesible. Ejemplo real: Delegar eventos en listas dinámicas.
  • L4. Compatibilidad y separación de capas (contenido/estilo/comportamiento)
    Separa correctamente HTML, CSS y JS. Ejemplo real: Cambiar el tema de la web sin tocar el HTML.

UD7. Comunicación asíncrona

¿Por qué es importante? La comunicación asíncrona permite crear aplicaciones web modernas y dinámicas.
RA/CE: Analizar mecanismos asíncronos, formatos, compatibilidad, uso de librerías.
Ejemplo real: Cargar datos de un servidor sin recargar la página o mostrar mensajes de error si falla la conexión.

  • L1. fetch, Promesas y async/await; estados y errores
    Realiza peticiones a servidores y gestiona respuestas. Ejemplo real: Mostrar noticias actualizadas desde una API.
  • L2. JSON y actualización dinámica del DOM
    Trabaja con datos en formato JSON y actualiza la web en tiempo real. Ejemplo real: Mostrar una lista de usuarios obtenida de un servidor.
  • L3. CORS, compatibilidad, librerías y pruebas
    Gestiona la seguridad y compatibilidad de tus peticiones. Ejemplo real: Usar una librería para consumir APIs externas.
